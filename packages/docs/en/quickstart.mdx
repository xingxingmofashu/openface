<Info>
  The repository is currently private to the workspace and the package is not published as a global binary yet, so the
  examples below run the CLI directly from source.
</Info>

## Prerequisites

- Bun `1.3.x` or newer
- Internet access the first time you download a model
- Enough disk space for local model caching under `~/.local/share/openface/models`

## Install dependencies

```bash
bun install
```

## Optional shortcut

If you do not want to repeat the long Bun command, create a local shell alias:

```bash
alias openface='bun run --cwd packages/openface --conditions=browser src/index.ts'
```

All command examples below use that alias. If you prefer not to define it, replace `openface` with the full Bun command.

## Inspect the CLI

```bash
openface --help
```

You should see three top-level commands:

- `translation`
- `text-generation`
- `pull`

## Try translation

```bash
openface translation --model Helsinki-NLP/opus-mt-zh-en
```

Then type inside the REPL:

```text
> 你好，世界
> .exit
```

<Tip>
  Translation uses a streamer internally, so interactive output is the best-supported path in the current implementation.
</Tip>

## Try text generation

```bash
openface text-generation --model Xenova/gpt2 --stream
```

Then type a prompt:

```text
> Once upon a time in Shanghai,
> .exit
```

<Warning>
  In the current CLI flow, `--stream` is the most reliable way to see generated text directly in the terminal during a
  REPL session.
</Warning>

## Pre-download a model

```bash
openface pull --model Helsinki-NLP/opus-mt-zh-en --task translation
```

This downloads the required model files into the configured local cache before you start a session.
