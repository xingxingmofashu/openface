<Info>
  The package is not published as a global binary yet. All examples below use the local launcher at
  `packages/openface/bin/openface`.
</Info>

## Prerequisites

- Bun `1.3.x` or newer
- Internet access for the first `pull`
- Enough disk space for models under `~/.local/share/openface/models`

## Install dependencies

```bash
bun install
```

## Inspect the CLI

```bash
packages/openface/bin/openface --help
```

You should see these top-level commands:

- `pull`
- `config`
- `list`
- `remove`
- `run`

## Optional shortcut

```bash
alias openface='./packages/openface/bin/openface'
```

All remaining examples use that alias.

## Optional build step

```bash
bun run build
```

This runs `packages/openface/scripts/build.ts` and writes the built CLI to `packages/openface/dist/openface`. The
launcher works without this step, but build verification is recommended before release work.

## Pull a model

Start with a translation model because `run` currently supports translation and text generation only.

```bash
openface pull Helsinki-NLP/opus-mt-zh-en
```

`pull` does two things:

- downloads the required files into the local cache
- records model metadata in `~/.config/openface/model.json`

## Run the model

```bash
openface run Helsinki-NLP/opus-mt-zh-en
```

Inside the REPL:

```text
> 你好，世界
> .help
> .exit
```

## Inspect config and cache

```bash
openface config list
openface list
```

`config list` shows the merged runtime configuration. `list` scans the configured cache directory and prints the models
found there.

## Remove a model

```bash
openface remove Helsinki-NLP/opus-mt-zh-en
```

## Notes

- `run` checks the pulled-model registry, not just the cache directory. If you skip `pull`, `run` will not know about
  the model.
- `pull` can download many Hugging Face task types, but `run` only creates handlers for `translation` and
  `text-generation`.
