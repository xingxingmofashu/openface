<Info>
  **Prerequisites**
  - Bun installed for the CLI project
  - Mint installed globally for the docs preview
</Info>

## Project commands

<Steps>
<Step title="Install dependencies">

```bash
bun install
```

This installs the root workspace and links the packages under `packages/*`.
</Step>

<Step title="Run the CLI locally">

```bash
bun dev
```

This runs the root `dev` script, which delegates to `packages/openface/src/index.ts`.

</Step>
<Step title="Build the CLI">

```bash
bun run build
```

This runs `packages/openface/scripts/build.ts` and writes the distributable CLI to `packages/openface/dist/openface`.

</Step>
<Step title="Type-check the workspace">

```bash
bun typecheck
```
</Step>
</Steps>

## Preview docs

From `packages/docs`:

```bash
mint dev
```

Mint will start a local preview, usually on `http://localhost:3000`.

## Validate docs

```bash
cd packages/docs
mint validate
```

## Useful checks

- `mint validate` checks the docs build in strict mode.
- `mint broken-links` scans internal and external links.
- `bun run build` verifies the CLI build path and launcher integration.
- `bun typecheck` verifies the TypeScript workspace.

## Repository layout

```text
.
├── packages/docs/          # Mintlify site
└── packages/openface/
    ├── bin/openface        # launcher
    ├── scripts/build.ts    # build script
    ├── dist/openface       # built CLI output
    └── src/
        ├── cli/
        │   ├── commands/   # yargs command modules
        │   └── utils/      # REPL and helper utilities
        ├── config/         # User config + model registry
        ├── tasks/          # Hub downloads and pipeline wrappers
        └── index.ts        # runtime entrypoint
```

## Docs files

The Mint site for this repository lives under `packages/docs/` and is configured through `packages/docs/docs.json`.

## Notes

- The root `package.json` exposes `bun dev`, `bun run build`, and `bun typecheck`.
- The CLI surface currently consists of `pull`, `run`, `list`, `remove`, and `config`.
- `run` is REPL-first and currently supports `translation` and `text-generation` model tasks.
