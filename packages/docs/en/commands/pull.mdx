---
title: "Pull"
description: "Download a Hugging Face model into the local cache and register it for later use."
---

## Synopsis

```bash
openface pull <modelId>
```

## Positionals

| Argument  | Required | Description                                                      |
| --------- | -------- | ---------------------------------------------------------------- |
| `modelId` | Yes      | Hugging Face repository ID such as `Helsinki-NLP/opus-mt-zh-en`. |

## Common examples

```bash
openface pull Helsinki-NLP/opus-mt-zh-en
openface pull Xenova/gpt2
openface pull Xenova/distilbert-base-uncased-finetuned-sst-2-english
```

## What `pull` does

`pull`:

- verifies the repository exists on Hugging Face Hub
- fetches model metadata from the Hub
- detects the task from the model metadata
- downloads the required tokenizer, processor, and model artifacts for that task
- stores the model metadata in `~/.config/openface/model.json`

The exact download set comes from `SUPPORTED_TASKS` in `packages/openface/src/tasks/pull/tasks.ts`.

## Supported tasks

`pull` understands all 25 Hugging Face task types listed below:

| Task                             | Type       |
| -------------------------------- | ---------- |
| `text-classification`            | text       |
| `token-classification`           | text       |
| `question-answering`             | text       |
| `fill-mask`                      | text       |
| `summarization`                  | text       |
| `translation`                    | text       |
| `text2text-generation`           | text       |
| `text-generation`                | text       |
| `zero-shot-classification`       | text       |
| `text-to-audio`                  | text       |
| `feature-extraction`             | text       |
| `audio-classification`           | audio      |
| `zero-shot-audio-classification` | multimodal |
| `automatic-speech-recognition`   | multimodal |
| `image-to-text`                  | multimodal |
| `image-classification`           | multimodal |
| `image-segmentation`             | multimodal |
| `zero-shot-image-classification` | multimodal |
| `object-detection`               | multimodal |
| `zero-shot-object-detection`     | multimodal |
| `document-question-answering`    | multimodal |
| `background-removal`             | image      |
| `image-to-image`                 | image      |
| `depth-estimation`               | image      |
| `image-feature-extraction`       | image      |

This list is wider than the current runtime support of `openface run`.

## Cache behavior

Downloaded artifacts are stored in the configured cache directory. By default that is:

```text
~/.local/share/openface/models
```

## Important note

Successfully pulling a model does not guarantee it can be run through `openface run`. Today, the REPL only has handler
implementations for `translation` and `text-generation`.
