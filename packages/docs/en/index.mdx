## Typical workflow

<Steps>
  <Step title="Install dependencies">
    Clone the repository and run `bun install` from the project root. This is a Bun workspace, so the root install
    covers both `packages/openface` and `packages/docs`.
  </Step>
  <Step title="Run the CLI locally">
    Use `bun run --cwd packages/openface --conditions=browser src/index.ts --help` to inspect the available commands.
  </Step>
  <Step title="Pull a model">
    Start with `pull` so the model is cached locally and recorded in `~/.config/openface/model.json`.
  </Step>
  <Step title="Run the model">
    Use `run <modelId>` to open the interactive REPL. If the pulled model is not a `translation` or
    `text-generation` model, `run` will reject it.
  </Step>
</Steps>

## Project shape

OpenFace uses a simple workspace layout inspired by Bun-style monorepos:

1. `packages/openface` contains the CLI package.
2. `packages/docs` contains the Mintlify docs package.
3. `packages/openface/src/index.ts` registers the CLI commands with `yargs`.
4. `packages/openface/src/cli/commands/*` defines command behavior.
5. `packages/openface/src/tasks/*` wraps Hugging Face Hub and Transformers work.
6. `packages/openface/src/config/*` loads user config and model registry files.
