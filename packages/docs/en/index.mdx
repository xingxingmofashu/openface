## What OpenFace does

  </Card>
  <Card title="Pull Models" icon="download" href="/en/commands/pull">

  </Card>

## Typical workflow

<Steps>
  <Step title="Install dependencies">
    Clone the repository and run `bun install` from the project root.
  </Step>
  <Step title="Run the CLI locally">
    Use `bun run --cwd packages/openface --conditions=browser src/index.ts --help` to inspect the available commands.
  </Step>
  <Step title="Download or stream a model">
    Start with `pull` if you want to warm the local cache first, then use `translation` or `text-generation` in REPL
    mode.
  </Step>
</Steps>

## Project shape

OpenFace uses a small but clean layering model:

1. `packages/openface/src/index.ts` registers the CLI commands with `yargs`.
2. `packages/openface/src/cli/commands/*` defines command flags and user-facing behavior.
3. `packages/openface/src/tasks/*` wraps Hugging Face Transformers pipelines and model downloads.
4. `packages/openface/src/config/*` loads a global config file and merges it with sensible defaults.
