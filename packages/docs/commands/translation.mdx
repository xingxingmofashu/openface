title: "Translation"
description: "Translate text between languages with a Hugging Face translation model."
---

## Synopsis

```bash
openface translation --model <model-id> [--src_lang <code>] [--tgt_lang <code>]
```

## Options

| Flag | Alias | Required | Description |
| --- | --- | --- | --- |
| `--model` | `-m` | Yes | Hugging Face model repository ID. |
| `--src_lang` | `-s` | No | Source language code. Defaults to `zho_Hans`. |
| `--tgt_lang` | `-t` | No | Target language code. Defaults to `eng_Latn`. |

## Recommended workflow

Start the command, then enter text in the interactive prompt:

```bash
openface translation --model Helsinki-NLP/opus-mt-zh-en
```

```text
> 你好，世界
> .exit
```

<Tip>
  The current implementation is best used in REPL mode. Translation output is streamed while the model is generating.
</Tip>

## Language codes

OpenFace ships a large language-code map in `packages/openface/src/tasks/translation/languages.ts`. The defaults are:

- `zho_Hans` for Simplified Chinese
- `eng_Latn` for English

Use explicit language codes when your selected model requires them, especially multilingual translation models.

## Example: multilingual model

```bash
openface translation \
  --model facebook/nllb-200-distilled-600M \
  --src_lang zho_Hans \
  --tgt_lang eng_Latn
```

## Notes

- The CLI help currently exposes a positional `translation` argument, but the interactive REPL flow is the best-supported
  path in the current code.
- Models are cached locally according to the global configuration described in the configuration reference.
